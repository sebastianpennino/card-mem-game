Memory Game

This is a simple project example only to show my abilities as a developer. I used a TDD approach, using Jasmine for unit testing the code, and using some common javascript patterns to organize the code. On the CSS side I decided to use Boostrap to speed up development

Requirements

  1. Create a deck of 30 cards with 15 pairs of matching items (cards can be whatever you'd like such a letter, number, word, etc)
  2. Arrange the cards in a grid randomly with their faces hidden from the player ("face down")
  3. The user will click a card to reveal ("flip") it
  4. The user will click to reveal a second card. If it matches the first card, they are removed and the player scores. If it does not match, then the cards are both "flipped" back over.
  5. Once all cards have been matched, allow the user to start a new game.

Self imposed requirements

  • Don't use any MVC framework but try to respect the MVC principle (if possible)
  • Use a TDD approach (do the test fail, do the code, correct and repeat)
  • Try to use plain javascript (not dependand on jQuery or underscore)
  • Use common patterns and best practices
  • If you have the time use JSDocs style comments
